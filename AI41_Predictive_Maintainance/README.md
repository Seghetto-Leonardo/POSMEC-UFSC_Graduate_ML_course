Here I present the analyses I developed with the AI4I 2020 Predictive Maintenance Dataset [Dataset]. (2020). UCI Machine Learning Repository. https://doi.org/10.24432/C5HS5C (for the dataset description, see either the original article which is referenced here or this link https://archive.ics.uci.edu/dataset/601/ai4i+2020+predictive+maintenance+dataset, which leads to the UC Irvine machine learning repository page correspondent to this dataset's description). My goal was to classify if the operating parameters represent a normal regime or a failure one (and also tried to classify the type of failure).

For preprocessing I dropped the non-numeric features UID e Product ID (since I judged them as irrelevant for my goal). Then transformed the Type feature from an letter class to numerical one via Label Encoding (being Low = 0. Medium = 1, High = 2. I introduced this due to the contributuion of this feature on the tool wear feature, which directly influences the machine failure by tool wear). Finally, I normalized all features via the StandartScaler function from numpy to avoid dependece on feature scale.

Instead of applying the classification models directly into the normalized features, I decided to transform the dataset from its current base to its Principal Components, as a way to focus the failure detection on the most variance directions instead of "raw" features.
